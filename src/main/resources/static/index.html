The New Boston API Docs

This project was created following the series Spring Boot with Kotlin & JUnit 5 on YouTube, from thenewboston channel .

Here you can see the list of endpoints available from this project, and all of them are under /api route and it's following the RESTFul principles.

Type Route Params Description Expected response
GET /api/hello None It's a simple Hello World example, just following the tutorial apart, but it will become the health check endpoint on a future refactoring, returning the 200 status code. A text with the setence: Hello World from Spring Boot! Now the curse had been broken.
GET /api/banks None It returns a list of all registered banks with status code 200.
GET /api/banks/{accountNumber} None The details of the bank with the given account number passed in resource path variable with status code 200. If an unknown account number is passed in, it returns a 404 status with the message Couldn't find a bank with account number accountNumberPassedIn
POST /api/banks Must send a body with the following json structure:
It creates a new bank with the given properties passed in the body. If is passed an accountNumber that already exists it will return the message: Bank with account number {accountNumber} already exists. and the status 400. This status is also sent if an invalid json is given. It will return the same body that was passed.
